Inside Macintosh

Reader is a thin C program. Almost everything visible on screen is drawn or managed by the classic Mac OS Toolbox. Three managers do most of the work.

QuickDraw

QuickDraw is the 2D drawing engine baked into every classic Macintosh. It owns rectangles, regions, the pen, fonts, and pixel manipulation. Reader calls TextFont, TextSize, and TextFace to switch styles, then MoveTo + DrawText to paint each word at its computed position. The bullet glyphs are just PaintOval calls into a small rect.

Window Manager

The Window Manager handles the document window itself: the title bar, the close box, dragging, growing, the front-to-back ordering. Reader's window is defined declaratively in a WIND resource and instantiated with GetNewWindow. The event loop's FindWindow turns a click point into a part code (in title bar? in content? in grow box?) so Reader knows whether to scroll, navigate, or hand the click back to the system.

Resource Manager

The Resource Manager is the original asset pipeline. Menus, windows, alerts, strings, and even the application icon all live in the resource fork as small typed records (MENU, WIND, ALRT, STR#). Reader's UI text and fallback HTML body come from a STR# resource at runtime via GetIndString, which means localizing the app would be a matter of swapping resources, not recompiling.
